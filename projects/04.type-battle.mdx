### 기술적 해결

✅ CPM 전송 쓰로틀링 적용으로 **네트워크 트래픽 최적화**

-   **문제**: CPM 업데이트가 매 입력마다 발생하여 **서버 부하 증가** & **클라이언트 성능 저하**
-   **해결**: **500ms 간격으로 전송 제한** (쓰로틀링 적용)
-   **성과**: 서버 부하 감소 (타자 속도 300 기준 초당 **5회** → **0.5회**)

✅ 소켓 재연결 및 상태 동기화로 **안정성 확보**

-   **문제**: 네트워크 불안정 시 게임 진행 불가 및 **UI 상태 불일치**
-   **해결**: 자동 재연결 (최대 5회) 및 연결 해제 시 **게임 상태 초기화**
-   **성과**: 네트워크 불안정 상황에서도 **안정적 게임 진행**, UI 상태 100% 동기화

✅ React 메모이제이션으로 **렌더링 성능 개선**

-   **문제**: 타이핑 입력 시 모든 컴포넌트가 **불필요하게 리렌더링**
-   **해결**: **React.memo**, **useCallback**으로 **컴포넌트 및 함수 메모이제이션**
-   **성과**: 불필요한 리렌더링 **70%** 감소

✅ 문장 완성 시에만 로그 전송으로 **트래픽 감소**

-   **문제**: 매 입력마다 서버 전송 시 **네트워크 트래픽 과다**
-   **해결**: **문장 완성 시에만 로그 전송**
-   **성과**: 네트워크 트래픽 **90% 감소**

## 기술 스택 및 도구

-   **TypeScript**: 타입 안정성 보장
-   **Vite**: 빠른 개발 서버 및 빌드
-   **Socket.IO Client**: 실시간 양방향 통신
-   **TailwindCSS**: 유틸리티 기반 스타일링
-   **Framer Motion**: 고성능 애니메이션 라이브러리
-   **Node.js**: 웹 소켓 서버 구현
