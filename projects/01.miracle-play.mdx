### 담당 업무

**페이지 내 다수의 API 엔드포인트 연동 작업 및 실시간 동기화**

-   토너먼트 정보, 참가자 목록, 실시간 랭킹, 유저 참가여부 등 **API 연동 작업**
-   토너먼트 상태별 **분기 처리** (토너먼트 시작 전/대기/시작/종료 등)
-   참가자 참여 여부에 따른 **UI 활성화/비활성화 작업**
-   **실시간 업데이트**를 위한 **폴링 최적화**

### 기술적 해결

**✅ 조건부 폴링을 통한 네트워크 요청 최적화**
- **문제**: 불필요한 상황에서도 지속적인 폴링으로 인한 **서버 부하** 및 **네트워크 비용 증가**
- **해결**: 토너먼트 상태에 따른 **조건부 폴링** 구현
  - 진행 중/모집 중 등 실시간 **동기화가 필요한 상태**에서만 **5초 간격 폴링 수행**
  - 종료 상태 및 시작 이전 단계에서는 폴링 자동 중단
- **효과**: **불필요한 API 요청 감소**

**✅ 타이머 컴포넌트 분리를 통한 렌더링 성능 개선**
- **문제**: 1초마다 업데이트되는 타이머로 인한 **전체 페이지 리렌더링 발생**
- **해결**: 타이머를 독립 컴포넌트로 분리하고 `React.memo`를 적용하여 **메모이제이션**
- **효과**: **불필요한 리렌더링 방지** 및 **페이지 성능능 최적화**

**✅ 백엔드 협업을 통한 API 설계 개선**
- **문제**: 모든 데이터를 반환하는 단일 API로 인한 **과도한 데이터 요청**
- **해결**: **백엔드 개발자와 협의**하여 **API를 용도별로 분리** 및 문서작성
  - 정적 데이터(토너먼트 기본 정보): 최초 1회만 요청
  - 동적 데이터(실시간 상태 정보): 필요 시에만 요청
- **효과**: API 요청 횟수 감소, **네트워크 비용 절감**

### 담당 업무

**토너먼트 생성 페이지의 상태 관리, 서버 통신 및 유효성 검사 구현**

- 토너먼트 생성 폼 유효성 검사 로직 구현
- 토너먼트 소요 시간 자동 계산 로직 개발
- 토너먼트 생성 API 연동 및 에러 핸들링 처리

### 기술적 해결

**✅ 과도한 프롭스 드릴링 해소**
- **문제**: 3단계로 구성된 토너먼트 생성 플로우에서 과도한 프롭스 드릴링으로 인한 **유지보수성 저하**
- **해결**: `redux toolkit`을 활용한 전역 상태 관리로 컴포넌트 간 데이터 전달 구조 개선
- **효과**: **코드 복잡도 감소** 및 **상태 관리 로직 중앙화**

**✅ 토너먼트 옵션 데이터 관리 체계화**
- **구현**: 토너먼트 설정 옵션을 상수 파일로 분리하여 관리
- **효과**: 
  - 옵션 추가/수정 시 단일 파일 관리로 작업 효율성 향상
  - 코드 가독성 및 재사용성 증대
  - 타입 안정성 확보 및 유지보수 용이성 개선

## 2️⃣ <u>모달 래핑 컴포넌트 도입</u>

-   중복 작성되던 모달의 기본 구조, 스타일, 공통 기능을 래핑 컴포넌트로 **분리**
-   모달 열림/닫힘 시 적용되던 스크롤 정지 기능을 개별 모달이 아닌 공통 관리로 **일원화**

---

## 3️⃣ <u>redux-toolkit을 사용한 서버 요청 감소 작업</u>

-   자주 변하지 않는 데이터(예: 유저 데이터) 및 여러 곳에서 사용되는
    데이터(예: 블록체인 코인 보유 수량 등)를 **Redux-toolkit** 전역 상태로 관리하여 **API 호출 최소화**
-   사용자 데이터, 설정값 등 반복 요청이 발생하던 리소스를 캐싱

### 성과

-   API 요청 횟수 **약 20% 감소**로 서버 부하 및 네트워크 비용 절감
-   코드 재활용성 증가로 **중복 로직 최소화** 및 유지보수성 향상

---

## 4️⃣ <u>커스텀 훅을 활용한 비즈니스 패턴 분리</u>

-   공통 API 요청 로직을 **커스텀 훅으로 분리**하여 재사용 가능하도록 설계
-   동일 API 요청이 필요한 여러 컴포넌트에서 훅 재활용

### 성과

-   **중복 코드 제거**로 코드라인 감소 및 유지보수성 향상
-   비즈니스 로직과 UI 로직 분리로 **코드 가독성 개선**
