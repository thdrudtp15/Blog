# Next.js에서의 폰트 최적화

웹 폰트를 적용하면 로딩 과정에서 FOUT(Flash of Unstyled Text) 현상이 발생하곤 합니다.
이 글에서는 Next.js에서 이러한 FOUT 현상을 어떻게 해결하고 최적화할 수 있는지 알아보겠습니다.

## 웹 폰트 로딩의 문제점

일반적으로 웹 폰트를 사용할 때는 다음과 같이 CDN을 통해 로드합니다.

```html
<link href="https://~~" rel="stylesheet" />
```

하지만 이 방식은 여러 문제점을 가지고 있습니다.

### FOUT (Flash of Unstyled Text)

시스템 폰트로 먼저 텍스트를 렌더링한 후, 웹 폰트가 준비되면 갑자기 폰트가 바뀌는 현상입니다.

**문제점:**

-   글자가 깜빡이며 전환됨
-   사용자가 읽던 위치를 놓칠 수 있음
-   전문적이지 않은 인상

### CLS (Cumulative Layout Shift)

폰트마다 글자의 크기와 간격이 모두 다르기 때문에, 웹 폰트가 로드되면 레이아웃이 변경되는 현상입니다.

**문제점:**

-   페이지 레이아웃이 갑자기 변경됨
-   SEO에 부정적 영향
-   Lighthouse 성능 점수 하락
-   사용자가 클릭하려던 버튼이 이동할 수 있음

### 외부 CDN 의존성

-   외부 CDN이 느리면 내 사이트도 느려짐
-   추가적인 네트워크 요청 발생
-   개인정보 보호 이슈 (사용자 IP 전송)

## 해결책: next/font

이러한 문제를 해결하기 위해 Next.js에서는 `next/font`를 제공합니다.

### next/font의 동작 방식

`next/font`는 빌드 타임에 다음과 같은 최적화를 수행합니다:

1. **빌드 시점에 폰트 다운로드**: 서버에서 직접 폰트를 제공
2. **자동 Self-Hosting**: `.next/static/fonts/` 디렉토리에 저장
3. **CSS size-adjust 자동 계산**: Fallback 폰트와 실제 폰트의 크기를 맞춤

**결과:**

-   CDN 의존성 제거
-   네트워크 요청 감소
-   더 빠른 로딩 속도
-   **CLS 0 달성**

## 적용 방법

### Google Fonts 사용

가장 기본적인 사용법은 다음과 같습니다.

```ts
// app/layout.tsx
import { Inter } from 'next/font/google';

const inter = Inter({
    subsets: ['latin'],
    display: 'swap',
});

export default function RootLayout({
    children,
}: {
    children: React.ReactNode;
}) {
    return (
        <html lang="ko" className={inter.className}>
            <body>{children}</body>
        </html>
    );
}
```

Google Fonts에서 제공하는 모든 폰트를 사용할 수 있으며, 자동으로 최적화됩니다.

### Local 폰트 사용

구입했거나 직접 제작한 커스텀 폰트의 경우 다음과 같이 사용할 수 있습니다.

```ts
// app/layout.tsx
import localFont from 'next/font/local';

const pretendard = localFont({
    src: [
        {
            path: '../public/fonts/Pretendard-Regular.woff2',
            weight: '400',
            style: 'normal',
        },
        {
            path: '../public/fonts/Pretendard-Bold.woff2',
            weight: '700',
            style: 'normal',
        },
    ],
    variable: '--font-pretendard',
    display: 'swap',
});

export default function RootLayout({
    children,
}: {
    children: React.ReactNode;
}) {
    return (
        <html lang="ko" className={pretendard.variable}>
            <body className="font-sans">{children}</body>
        </html>
    );
}
```

**주의:** 폰트 파일은 반드시 `public/fonts` 디렉토리에 위치해야 합니다.

**프로젝트 구조:**

```
project/
├── app/
│   └── layout.tsx
└── public/
    └── fonts/
        ├── Pretendard-Regular.woff2
        └── Pretendard-Bold.woff2
```

### 특정 컴포넌트에만 적용

전역이 아닌 특정 컴포넌트에만 폰트를 적용할 수도 있습니다.

```ts
// components/Header.tsx
import { Inter } from 'next/font/google';

const inter = Inter({
    subsets: ['latin'],
    weight: '700',
    display: 'swap',
});

export default function Header() {
    return (
        <header>
            {/* Logo에만 특별한 폰트 적용 */}
            <h1 className={inter.className}>Logo</h1>
            <nav>{/* 기본 폰트 사용 */}</nav>
        </header>
    );
}
```

## 폰트 속성 상세 설명

```ts
const inter = Inter({
    subsets: ['latin'],
    display: 'swap',
    weight: ['400', '700'],
    variable: '--font-inter',
    preload: true,
});
```

### subsets

폰트에서 사용할 문자 집합을 지정합니다.

```ts
subsets: ['latin']; // 영문, 숫자, 기본 특수문자
```

-   한글 폰트의 경우 `['latin']`만 명시해도 한글이 자동으로 포함됩니다.
-   필요한 문자만 포함하여 폰트 파일 용량을 최적화합니다.

### display

폰트 로딩 방식을 결정하는 중요한 속성입니다.

```ts
display: 'swap'; // 권장
```

**옵션:**

-   **`swap` (권장)**: 폰트 로드 전 fallback 폰트를 즉시 표시하고, 로드 완료 시 웹 폰트로 교체
-   `optional`: 폰트 로드 실패 시 fallback 폰트 유지
-   `block`: 폰트 로드까지 텍스트를 숨김 (비권장)
-   `fallback`: swap과 block의 중간

대부분의 경우 `swap`을 사용하는 것이 좋습니다.

### weight (선택사항)

사용할 폰트의 굵기를 지정합니다.

```ts
// 특정 굵기만 로드 (용량 최적화)
weight: ['400', '700'];

// 모든 굵기 지원 (Variable Font)
weight: 'variable';
```

필요한 굵기만 지정하면 폰트 파일 용량을 크게 줄일 수 있습니다.

**Variable Font 장점:**

-   하나의 파일로 모든 굵기 (100~900) 사용 가능
-   더 효율적인 용량
-   중간 굵기도 사용 가능 (예: `font-weight: 450`)

### variable (선택사항)

CSS 변수명을 지정합니다. Tailwind CSS와 함께 사용할 때 특히 유용합니다.

```ts
variable: '--font-inter';
```

**사용 예시:**

```tsx
// layout.tsx
<html lang="ko" className={inter.variable}>
    <body className="font-sans">{children}</body>
</html>
```

```js
// tailwind.config.js
module.exports = {
    theme: {
        extend: {
            fontFamily: {
                sans: ['var(--font-inter)', 'sans-serif'],
            },
        },
    },
};
```

### preload (선택사항)

폰트를 우선적으로 로드할지 결정합니다.

```ts
preload: true; // 기본값, 폰트를 우선 로드
preload: false; // 필요할 때만 로드
```

**사용 가이드:**

-   메인 폰트: `preload: true` (기본값, 권장)
-   장식용 폰트: `preload: false` (필요시에만 로드)
