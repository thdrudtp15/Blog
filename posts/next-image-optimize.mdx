# Next.js의 Image 컴포넌트 가이드

Next.js에서 제공하는 Image 컴포넌트의 사용법과 최적화 기능들에 대해서 작성합니다.

## Next.js의 Image 컴포넌트란?

Next.js에서 기본적으로 제공하며, **자동 최적화** 등 유용한 기능을 제공합니다.

Next.js에서는 `<img/>` 태그보단 **Image 컴포넌트**를 사용하여 이미지를 렌더링하라고 권장하고 있습니다.

```ts
import Image from 'next/image';

export default function Page() {
    return (
        <Image
            src="/profile.png"
            width={500}
            height={500}
            alt="Picture of the author"
        />
    );
}
```

## 이미지 최적화 속성 및 사용 방법

Image 컴포넌트엔 다양한 속성들이 있지만, 해당 글에서는 **최적화와 관련된 속성**만을 다룹니다.

최적화 속성을 제외한 속성들에 대해 알아보고 싶으시다면 [공식 문서](https://nextjs.org/docs/pages/api-reference/components/image)를 참조 부탁드립니다!

---

### sizes

```ts
// 뷰포트 768px에서 100vw 차지, 768px 보다 넓을 경우 50vw로 렌더링
<Image ... sizes="(max-width: 768px) 100vw, 50vw"/>
```

**반응형 이미지 최적화**의 핵심 속성입니다.

뷰포트의 너비별로 이미지가 차지할 크기의 **힌트를 제공**하게 됩니다.

반응형 작업을 하게 될 경우 grid가 변경되는 등 레이아웃 변경으로 이미지 공간이 차지하는 크기가 변하는 경우가 있는데, 해당 속성으로 이미지의 크기를 정해 Image 컴포넌트에 힌트를 제공해줄 수 있고 이 힌트를 통해 **브라우저가 적절한 이미지의 크기를 선택**할 수 있습니다.

---

### quality

```ts
// 40퍼센트의 퀄리티로 렌더링
<Image ... quality={40}/>
```

속성명처럼 **이미지의 품질을 결정**하는 역할을 하는 속성입니다.

이미지가 표현되는 크기에 비해 제공하는 이미지의 용량이 지나치게 큰 경우가 있는데요, 이런 경우 유용하게 사용할 수 있는 속성입니다.

품질이 저하되는 단점이 있지만, 품질이 저하된 만큼 **파일의 크기 또한 작아져** 페인트 속도를 끌어올릴 수 있습니다.

---

### priority

```ts
// 즉시 로드
<Image ... priority={true}/>
```

필요할 때 로드하는 **lazy loading을 비활성화**하고 페이지 로드 시 **즉시 이미지를 로드**하도록 하는 속성입니다.

특히 페이지 로드 시 즉시 보여져야 하는 헤더의 로고, 히어로 섹션 이미지에 사용되며, 과도한 사용은 오히려 성능 저하를 불러 일으키기 때문에 **선택적으로 사용**하면 사용자 경험을 끌어올릴 수 있습니다.

또한 페이지 로드 시 **최우선으로 로드**하도록 설정됩니다.

**LCP 최적화**에 큰 도움이 됩니다.

> **LCP(Largest Contentful Paint)란?**
>
> 화면에 중요한(큰 배너 이미지, 주요 텍스트 등) 콘텐츠가 실제로 그려질 때까지의 시간을 의미합니다.

---

### loading

```ts
// 이미지가 뷰포트에 나타나게 될 때 이미지를 로드합니다 (이미지 지연)
<Image ... loading="lazy"/>
```

**이미지 로드의 시점을 결정**하는 속성입니다.

기본 값은 `lazy`로 Image 컴포넌트는 기본적으로 **lazy loading 형식**으로 이미지를 로드한다는 것을 알 수 있습니다.

위의 예시에서는 `lazy`를 이용해 뷰포트에 나타날 때에 로드를 하는 옵션을 사용했는데요, `eager`를 사용해 이미지가 **즉시 로드**되도록 설정할 수 있습니다.

`eager`의 경우 `priority`와 비슷하게 보일 수 있지만, 우선순위가 **최우선으로 설정되는 priority**와 달리 **로드 우선순위가 별도로 설정되지는 않습니다**.

---

### placeholder

```ts
// 이미지 로드 중 흐린 이미지 처리
<Image ... placeholder="blur"/>
```

이미지가 로드되기 전, **저해상도 블러 이미지**를 임시로 보여줄 수 있습니다.

**로컬 이미지 파일**을 사용하는 경우 빌드 시 자동으로 블러 이미지를 생성해줍니다.

**외부 URL**이나 Image의 src를 문자열로 제공하는 경우blur 사용 시에는 밑에서 설명드릴 `blurDataURL`이 필수적으로 필요합니다.

---

### blurDataURL

```ts
<Image placeholder="blur" blurDataURL="https://~~" />
```

**블러 이미지를 설정**합니다.

`placeholder="blur"`와 함께 사용되며, 주로 **Base64로 인코딩된 저해상도 이미지 데이터**를 제공합니다.

-   **로컬 이미지**: 빌드 시 자동 생성되므로 불필요
-   **외부 URL**: 필수 속성 (blurDataURL 미제공 시 blur 효과 미작동)

---

### loader

```ts
import Image from 'next/image';

const imageLoader = ({ src, width, quality }) => {
    return `https://example.com/${src}?w=${width}&q=${quality || 75}`;
};

const Page = () => {
    return (
        <Image
            loader={imageLoader}
            src="me.png"
            alt="Picture of the author"
            width={500}
            height={500}
        />
    );
};
export default Page;
```

**외부 이미지 CDN**(S3, Cloudinary 등)의 최적화 기능을 활용할 수 있는 속성입니다.

loader 속성에는 **사용자가 정의한 함수**를 전달하게 됩니다.

**함수 파라미터:**

-   `src`: Image 컴포넌트에 전달된 이미지 경로
-   `width`: Image 컴포넌트의 width 값 또는 자동 계산된 너비
-   `quality`: Image 컴포넌트의 quality 값 (기본값 75)

이미지 CDN에서 **쿼리 파라미터를 통한 크기/품질 조정**을 지원하는 경우, 이 함수를 통해 CDN에 최적화된 크기의 이미지를 요청할 수 있어 **사용자가 설정한 크기에 맞는 이미지 데이터**를 가져올 수 있습니다.

#### 전역 설정 예시

개별 이미지마다 loader를 설정하는 대신, **전역으로 설정**할 수도 있습니다:

```ts
// next.config.ts
module.exports = {
    images: {
        loader: 'custom',
        loaderFile: './my-loader.js',
    },
};
```

```ts
// my-loader.ts
const myLoader = ({ src, width, quality }) => {
    return `https://my-cdn.com${src}?w=${width}&q=${quality || 75}`;
};
export default myLoader;
```

이렇게 전역 설정을 하면 개별 Image 컴포넌트에 loader 속성을 지정하지 않아도 **자동으로 해당 함수가 적용**됩니다.
